# Coffendi instant storefront guidance

## Project map

- `src/App.jsx` contains the storefront router, shared shell, cart, retail pages, bulk brief, education, and sustainability experiences.
- `src/storefrontData.js` is the source of truth for the three instant-coffee products and optional public price display.
- `src/styles.css` is the coffee-toned design system and responsive stylesheet.
- `assets/instant/` contains editable image sources; `public/images/instant-*.webp` is generated by `npm run optimize:images`.
- `api/checkout.js` creates allowlisted Stripe Checkout sessions; `api/checkout-session.js` verifies success redirects; `api/stripe-webhook.js` verifies signed payment events and records fulfillment input. Never accept browser-supplied prices or expose secret keys.
- `api/inquiries.js` and `server/submissions.js` handle protected bulk-request persistence.
- The old green-coffee/Makendi files remain as unimported reference source. Do not reconnect or regenerate them unless a task explicitly asks for the legacy platform.

## Working conventions

- Use React 19-compatible JSX and preserve the storefront routes, cart persistence, and checkout boundary.
- Reuse the existing color, spacing, typography, button, form, and card primitives before adding a new visual pattern.
- Keep the instant-coffee aesthetic: cream and oat surfaces, espresso and cacao foundations, restrained caramel accents, tactile product photography, strong serif display type, and clear retail/commercial hierarchy.
- Keep desktop and 390px mobile layouts first-class. Avoid horizontal overflow and keep touch targets at least 44px where practical.
- Preserve semantic landmarks, visible keyboard focus, accessible names, reduced-motion behavior, and sufficient color contrast.
- Use `fetchPriority` rather than lowercase `fetchpriority` in JSX. Lazy-load non-hero imagery.
- Do not invent prices, pack sizes, inventory, delivery promises, certifications, quantified sustainability claims, refund terms, or legal policies when source data is missing.
- Keep retail cart/hosted checkout and commercial bulk inquiry as separate flows.
- Preserve unrelated working-tree changes. Do not regenerate optimized images or the sitemap unless the task requires it.

## Verification

- Run `npm run build` after source or dependency changes.
- Run `npm run test:checkout` after cart, checkout, product ID, or payment-configuration changes.
- Run `npm run test:a11y` with the Vite server available at `http://127.0.0.1:4173` after UI changes.
- Run `npm run build`, start `npm run preview -- --port 4173`, then run `npm run test:performance` against that production preview after changing layout, loading behavior, imagery, routing, or dependencies.
- Run `npm run test:bundle` after JavaScript, CSS, route, or dependency changes to preserve the compressed asset budgets.
- Run `npm run test:visual` with the same server after layout, navigation, cart, product, checkout, bulk-form, or responsive changes.
- Run `npm run test:production` after deployment, domain, security-header, readiness, health, sitemap, robots, or Vercel configuration changes. The check is read-only and targets `https://coffendi.vercel.app` by default.
- Run `git diff --check` before handoff.
- `npm run test:api` requires the linked private Vercel environment and exercises persistent submission APIs; do not run it unless that external test is explicitly in scope.

Done means the requested behavior works, the production build passes, relevant focused tests pass, and browser checks show no console errors, accessibility regressions, or horizontal overflow.
